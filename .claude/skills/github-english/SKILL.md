# GitHub artifacts in English

All text that ends up on GitHub must be written in English, regardless of the
conversation language:

- Issue titles and bodies
- Pull request titles, bodies, and review comments
- Commit messages
- Repository docs referenced from GitHub (README sections, etc.) unless the
  file is already in Japanese

Conversation replies to the user stay in Japanese (or whatever language the
user is using) — this rule only covers what gets published to GitHub.

When editing an existing artifact that is in Japanese, translate it to English
rather than appending English text alongside it.
